# Worklog Obsidian Plugin

Worklog 是一个本地化工时管理插件，用于在 Obsidian 中维护月度任务、每日工时、日历完成状态和年度统计看板。

## 当前能力

- 打开当月工时工作台，没有当月数据时自动创建
- 选择年月打开对应月份的工时工作台
- 年度工时看板按“上方总数据统计、下方全年任务类型饼图 + 每月计划/实际折线图”展示年度统计
- 使用本地 JSON 保存月度数据
- 可在插件设置中开启月度笔记创建，开启后实时同步看板、分类工时、日历、任务清单和每日工时明细
- 可在插件设置中维护任务类型和默认任务模板
- 新月份按启用的默认任务模板自动创建任务；没有模板时不会自动生成任务
- 新增任务，任务 ID 自动生成且不可自定义
- 新增工时，同一天同任务 ID 防重复
- 月度数据看板，包含统计卡和计划/实际柱状图
- 日历按实际月份天数渲染
- 手动勾选日期完成状态
- 任务状态下拉修改

## 启用方式

1. 确认插件目录位于：

```text
.obsidian/plugins/worklog/
```

2. 重启 Obsidian 或在社区插件里刷新插件列表。
3. 启用 `Worklog` 插件。
4. 使用左侧日历按钮，或命令面板执行：

```text
打开工时工作台
选择年月打开工时工作台
打开年度工时看板
```

## 数据路径

插件配置路径：

```text
.obsidian/plugins/worklog/config.json
```

默认数据路径：

```text
.obsidian/plugins/worklog/data/YYYY-MM.json
```

默认月度笔记路径：

```text
worklog/YYYY-MM.md
```

可以在插件设置里修改目录。默认数据目录位于 `.obsidian` 下，不会显示在 Obsidian 文件列表中；如果你的同步工具不包含 `.obsidian`，可以把数据目录改到库内普通目录。月度笔记默认不创建；需要笔记同步时，在 Worklog 设置中开启“新建月度笔记并实时同步数据”。

首次启用时，任务类型和默认任务模板为空，需要先在 Worklog 设置中手动新增。

## 初始化

初始化插件数据时，只需要删除 `config.json` 和数据目录下的月份 JSON。不要删除插件目录中的 `main.js`、`styles.css`、`manifest.json` 和 `README.md`。

## 说明

插件数据保存在当前库内，不依赖 Components、QuickAdd 或外部服务。
